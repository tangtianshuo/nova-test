# Plan 06：报告与导出计划

## 目标

- 生成可回放的报告（摘要/缺陷/轨迹）
- 支持导出 JSON（P0），HTML/PDF（P1）
- 以 Golden Files 的方式做回归保护（Harness）

## 依赖关系

| 依赖项 | 说明 |
|--------|------|
| EP-02 (数据库) | 从数据库读取 Instance/Steps/Defects 数据 |
| EP-04 (执行面 Steps) | 报告需要聚合步骤执行结果与状态 |
| EP-05 (HIL 记录) | 报告需要关联 Human-in-the-Loop 决策记录 |

## 范围

### 报告（P0）

- 从 Instance/Steps/Defects/HIL 记录生成 Report
- 报告与步骤可互相跳转

### 导出（P0/P1）

- P0：导出 JSON（ReportSchema，含 schema_version）
- P1：导出 HTML/PDF

## Harness（先做）

### H1：ReportSchema Harness（P0）

- 输出必须符合 ReportSchema

**测试用例：**

- `should_generate_valid_report_schema` - 验证生成的报告符合 ReportSchema 定义
- `should_include_all_steps_in_report` - 验证报告中包含所有执行步骤
- `should_aggregate_defects_correctly` - 验证缺陷信息正确聚合到报告中
- `should_link_hil_decisions` - 验证 HIL 决策记录正确关联到报告

### H2：Golden File Harness（P0）

- 固定输入 steps fixtures，导出 JSON 必须与 golden 文件一致
- 变更时：需要显式更新 golden 并记录原因

**测试用例：**

- `should_match_golden_json_on_fixed_input` - 固定输入下导出 JSON 必须与 golden 文件完全一致
- `should_detect_breaking_change` - 检测到输出变化时测试应失败
- `should_require_explicit_update_on_change` - 变更时需要显式更新 golden 文件并记录原因

### H3：Security Harness（P0）

- 导出内容不得包含敏感字段（token/密码/API key）

**测试用例：**

- `should_not_include_token_in_export` - 导出内容不得包含 token 字段
- `should_not_include_password_in_export` - 导出内容不得包含 password 字段
- `should_not_include_api_key_in_export` - 导出内容不得包含 api_key 字段
- `should_scan_for_sensitive_patterns` - 扫描导出内容中的敏感信息模式（如 Bearer、secret 等）

## 设计原则（SOLID）

- ReportGenerator 只负责聚合，不负责存储/下载（SRP）
- Exporter 采用接口隔离：JsonExporter/HtmlExporter/PdfExporter（ISP/OCP）

- 导出模块
- Harness：Schema + golden + security

## 验收标准
