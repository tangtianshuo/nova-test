# 中文注释规范

## 1. 注释原则

- **关键业务逻辑必须有中文注释**
- **复杂算法必须有中文注释说明意图**
- **安全相关代码必须有中文注释说明风险**
- **HIL 分支必须有中文注释说明决策原因**

---

```typescript
// 4. 验证通过 -> 继续执行

}
```

```typescript
  // ...

```

### HIL 分支处理

```typescript
// 风险: modify 需要验证修改后的参数合法性

}
```

## 3. 注释格式规范

```typescript
/**
 * 函数/模块简要说明
 * 
 * @param param1 - 参数说明
 * @returns 返回值说明
 * @throws 可能抛出的异常
 * 
 * 关键决策点/风险说明
 */
```

---

## 4. 禁止事项

| 禁止项 | 说明 |
|--------|------|
| 无意义注释 | 禁止 `// 设置 i 的值` 这类显而易见的注释 |
| 注释与代码不一致 | 修改代码时必须同步更新注释 |
| 暴露敏感信息 | 禁止在注释中出现密钥、密码、内部 IP 等 |
