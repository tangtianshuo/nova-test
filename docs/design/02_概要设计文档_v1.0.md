# 黑盒自动化测试工具 - 概要设计文档 (v1.0)

## 1. 架构设计原则

基于 **AaaS (Agent as a Service)** 概念，系统整体架构采用 **微服务与云原生架构**。通过控制面（Control Plane）与执行面（Execution Plane）分离的设计，实现高可用、高并发和租户安全隔离。系统严格遵循 **SDD (Schema-Driven Development)** 规范，所有服务间通信均基于明确定义的接口和数据 Schema。

## 2. 现代技术栈选型

| 层次 | 技术选型 | 说明 |
| --- | --- | --- |
| **前端 (Web UI)** | Next.js (React) + TailwindCSS + Zustand | 现代化的SaaS控制台，支持SSR及快速响应。WebSocket用于实时接收Agent截图和日志流。 |
| **后端 API 网关 / 控制面** | NestJS (Node.js) + GraphQL/REST | 处理租户管理、RBAC权限、计费、任务调度分发。使用 TypeScript 提供极佳的 Schema 支持。 |
| **AaaS 执行引擎 / 数据面** | FastAPI (Python) + LangGraph | FastAPI提供高性能的Agent触发接口，LangGraph负责维护状态机流转。 |
| **视觉与浏览器基建** | Fara-7B + Magentic-UI (Playwright) | Fara-7B作为视觉大脑（可部署在GPU节点/NPU端侧），Magentic-UI提供沙盒内的无头浏览器控制。 |
| **数据库与缓存** | PostgreSQL + Redis + MinIO (S3) | PostgreSQL存业务数据与状态，Redis作任务队列与缓存，MinIO存储截图和视频轨迹。 |
| **基础设施** | Docker + Kubernetes (K8s) | 实现Agent环境的弹性伸缩、隔离及沙箱逃逸防护。 |

## 4. 核心子系统设计

### 4.1 SaaS 控制面板 (Control Plane)
控制面板作为系统的业务大脑，负责与用户的交互。
- **租户隔离**：使用 PostgreSQL 的 Row-Level Security (RLS) 或基于 `tenant_id` 的应用层逻辑进行数据隔离。
- **任务队列**：将用户的测试请求（如“测试登录到购买的全流程”）封装为标准的 `TestTaskSchema`，投入 Redis 队列（如 BullMQ）。
- **实时协同桥梁**：维护与浏览器的 WebSocket 连接，当 AaaS 引擎触发 `Human-in-the-Loop (HIL)` 中断时，将现场截图及决策请求推送到前端 UI，等待人工响应后将决策结果写回引擎。

### 4.2 AaaS 执行引擎 (Execution Plane)
以 Agent 的形式对外提供服务（Agent as a Service）。
- **Worker 节点**：监听 Redis 队列，一旦拉取到任务，在 K8s 中启动或分配一个隔离的 Pod（内含 Magentic-UI 浏览器沙箱）。
- **状态机编排**：由 LangGraph 实例控制整个生命周期。每个步骤的状态变化都遵循 `AgentStateSchema`。
- **视觉决策**：通过 RPC/HTTP 调用 Fara-7B 推理服务（可剥离为独立的GPU集群），输入当前屏幕截图与历史上下文，输出具体坐标和操作类型。

### 4.3 存储与审计
- **结构化数据**：租户、用户、任务配置、执行记录、资产模板存储于 PostgreSQL。
- **非结构化数据**：测试过程中的截屏、录屏存储于 S3 协议对象存储（如 MinIO），通过预签名URL供前端访问。
- **状态持久化**：LangGraph 的 Checkpointer 状态序列化后存入 PostgreSQL，以支持断点续测和容错恢复。

## 5. 扩展性与高可用设计

1. **计算与业务解耦**：将资源消耗极大的 Fara-7B 推理与 LangGraph 编排分离，GPU节点独立部署并基于负载进行HPA（Horizontal Pod Autoscaling）。
2. **沙箱逃逸防护**：每个 Magentic-UI 实例在独立的 Docker 容器内运行，配置最小化权限，无宿主机文件系统访问权限，测试结束后销毁环境，保障安全性。
