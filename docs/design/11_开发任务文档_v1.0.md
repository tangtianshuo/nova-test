# 黑盒自动化测试工具 - 开发任务文档 (v1.0)

## 1. 说明

本任务拆解以 `10_详细需求文档_v1.0.md` 为输入，输出可执行的研发任务集。

### 1.1 任务分层
- `Epic`：可交付的业务能力域（跨前后端/基础设施）
- `Feature`：面向用户的功能集合
- `Task`：开发可执行单元（可在 0.5~3 天内完成）

### 1.2 通用完成标准（DoD）
- 通过 Schema 校验（SDD）：请求/响应/事件至少有结构定义与版本字段
- 关键路径有最少可用的审计字段（操作者/时间/原因）
- 有最小可运行的端到端链路验证（创建任务 -> 启动实例 -> 产生 Step -> 触发/处理 HIL -> 生成报告）

## 2. 里程碑（建议）

### M0：SDD 基线（1 周）
- 产出：核心对象 Schema、数据库初始化、接口骨架

### M1：MVP 可用闭环（3~4 周）
- 产出：SaaS 基础（租户/用户/任务/实例），AaaS 引擎最小闭环（循环执行 + 推流 + HIL 批准/修改 + 报告）

### M2：V1 上线增强（3~4 周）
- 产出：模板库、定时触发、导出、断点续测、可观测性与审计完善

## 3. Epic 拆解

### EP-01 SDD 与契约基建

#### FE-01-01 核心 Schema 定义
- T-01-01 定义 `TaskSchema`（输入/输出/constraints/version）
- T-01-02 定义 `AgentStateSchema`（对齐 LangGraph 状态）
- T-01-03 定义 `ActionSchema` 与 `EventSchema`（SCREENSHOT_UPDATE/AGENT_THOUGHT/HIL_REQUEST）
- T-01-04 定义 `ReportSchema`（摘要/缺陷/轨迹/导出）

#### FE-01-02 契约落地与校验
- T-01-05 在后端引入 Schema 校验（请求/响应）并生成示例
- T-01-06 建立 Schema 版本策略（兼容/破坏性变更规则）

依赖：无（起点）。
交付物：Schema 文件/示例 payload/版本策略说明。

### EP-02 数据库与存储（PostgreSQL/Redis/S3）

#### FE-02-01 数据库初始化
- T-02-01 建立 `Tenant/User/TestTask/AgentInstance/TestStep/HilTicket` 表结构
- T-02-02 建立基础索引与查询策略（按 tenant_id、task_id、instance_id）
- T-02-03 定义数据保留期字段与可扩展配置（用于 NFR-SEC-001）

#### FE-02-02 对象存储与引用
- T-02-04 接入 S3/MinIO 客户端与预签名下载策略
- T-02-05 Step 写入 `screenshotUrl`/（可选）`videoUrl`

#### FE-02-03 Redis 队列与 PubSub
- T-02-06 建立任务队列键空间（`queue:agent_tasks`）
- T-02-07 建立推流频道约定（`pubsub:agent_stream:<instance_id>`）

依赖：EP-01（对象 Schema）。
交付物：可运行的数据库迁移/初始化脚本与基础数据访问层。

### EP-03 SaaS 控制面（鉴权、任务、实例）

#### FE-03-01 鉴权与 RBAC
- T-03-01 实现 JWT 登录/鉴权中间件
- T-03-02 实现 RBAC（至少 4 角色）并覆盖核心接口
- T-03-03 实现 Workspace 切换与 tenant 上下文

#### FE-03-02 任务与实例 API
- T-03-04 实现 `POST /tasks`、`PATCH /tasks/{id}`、`GET /tasks`
- T-03-05 实现 `POST /tasks/{id}/instances`、`GET /instances/{id}/state`
- T-03-06 实现实例列表与筛选（状态、时间、创建人）

依赖：EP-01、EP-02。
交付物：可对外使用的控制面 API（最小闭环）。

### EP-04 AaaS 执行面（LangGraph + Magentic-UI + Fara-7B）

#### FE-04-01 Worker 基础与沙箱
- T-04-01 Worker 拉取队列任务并创建 Instance 运行上下文
- T-04-02 启动/复用浏览器沙箱（Magentic-UI/Playwright）并抓取截图

#### FE-04-02 状态机实现
- T-04-03 实现 LangGraph 节点：init/explore/check_hil/execute/verify
- T-04-04 实现 `max_steps` 与 `end` 终止逻辑
- T-04-05 将 Step 逐步落库（截图URL、action、验证结论）

#### FE-04-03 模型集成与输出约束
- T-04-06 对接 Fara-7B 推理服务（本地/远端）
- T-04-07 强制 JSON 输出并做容错（解析失败 -> 重试/触发 HIL）

依赖：EP-01、EP-02、EP-03（Instance/Task 基础）。
交付物：可运行的 AaaS 最小循环执行能力。

### EP-05 实时推流与 HIL

#### FE-05-01 WebSocket 推流
- T-05-01 建立 `wss /ws/instances/{id}/stream` 服务端与鉴权
- T-05-02 推送事件：SCREENSHOT_UPDATE/AGENT_THOUGHT/HIL_REQUEST

#### FE-05-02 HIL 工单
- T-05-03 创建工单：命中规则时落库并变更 instance 状态为 `PAUSED_HIL`
- T-05-04 实现 `POST /instances/{id}/resume`（approve/reject/modify）
- T-05-05 HIL 处理后的状态恢复与继续执行

#### FE-05-03 人工接管（P1）
- T-05-06 接管模式的输入通道与操作记录落库（形成 Step）
- T-05-07 交还给 Agent 后继续执行

依赖：EP-04。
交付物：可观测、可接管的实例执行体验。

### EP-06 报告、资产与导出

#### FE-06-01 报告生成
- T-06-01 Instance 结束后生成报告摘要与缺陷列表
- T-06-02 报告详情支持 Step 跳转与回放

#### FE-06-02 导出（P1）
- T-06-03 导出 JSON（对齐 ReportSchema，供 CI 解析）
- T-06-04 导出 HTML/PDF（可延后到 P1）

依赖：EP-02、EP-04、EP-05。
交付物：可交付给研发/管理的可读报告与资产。

### EP-07 前端控制台（UI/UE 落地）

#### FE-07-01 基础框架与视觉体系
- T-07-01 落地 UI 设计系统（颜色/间距/组件变体）
- T-07-02 基础布局：导航、Workspace、消息中心

#### FE-07-02 任务/实例/工单页面
- T-07-03 任务列表与创建向导（Wizard 4 步）
- T-07-04 实例详情 Live Console（截图 + 动作卡 + Timeline + Log）
- T-07-05 HIL 工单列表与处理面板（approve/reject/modify）

#### FE-07-03 报告与回放
- T-07-06 报告列表与详情（摘要/缺陷/回放）
- T-07-07 导出入口与下载体验

依赖：EP-03、EP-05、EP-06。
交付物：可用的 SaaS 控制台与实时执行体验。

### EP-08 安全、审计与可观测性（上线门槛）

#### FE-08-01 安全基线
- T-08-01 敏感信息处理：UI 掩码 + 不进入模型上下文
- T-08-02 沙箱网络隔离策略与域名黑名单

#### FE-08-02 审计与追溯
- T-08-03 关键操作审计：终止、批准高危、修改动作
- T-08-04 Instance/Step 关联链路可追溯（trace_id/instance_id）

#### FE-08-03 可观测性（P1）
- T-08-05 指标：步数、耗时、失败率、HIL 次数、推流断开率
- T-08-06 日志与告警：实例失败、重试过多、额度不足

依赖：全局。
交付物：生产可用的安全与运维基础。
