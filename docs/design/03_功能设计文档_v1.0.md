# 黑盒自动化测试工具 - 功能设计文档 (v1.0)

本功能设计基于 SaaS 模式与 AaaS (Agent as a Service) 概念进行拆解，主要划分为五大功能模块。

## 1. 租户与权限管理模块 (Tenant & IAM)

为满足 SaaS 化要求，系统提供完备的多租户体系。

### 1.1 租户 (Workspace) 管理
- **创建与初始化**：注册后自动创建租户空间，支持自定义企业 Logo、名称。
- **资源与额度管理**：显示当前套餐（Free / Pro / Enterprise），实时展示 Agent 运行时长、模型 Token 消耗量、剩余额度。
- **账单与订阅**：支持按量付费或包月订阅的账单生成与支付接口对接（如 Stripe）。

### 1.2 成员与 RBAC 权限
- **成员邀请**：通过链接或邮箱邀请团队成员加入 Workspace。
- **角色分配**：内置 Admin、Architect、Tester、Viewer 等角色。
- **细粒度权限**：基于 SDD 规范的权限矩阵，精确控制到 API 接口和页面按钮级别。

---

## 2. 自动化测试任务管理模块

提供测试生命周期的核心管理功能。

### 2.1 任务编排与配置 (Task Configuration)
- **自然语言用例输入**：支持通过自然语言直接描述测试目的（例如：“登录系统，搜索‘笔记本电脑’，将其加入购物车并校验购物车数量加一”）。
- **约束条件设置**：设定测试边界，如“禁止进行实际的支付操作”、“最大探索步数50步”。
- **环境变量配置**：配置测试所需的测试账号池、目标环境URL等。

### 2.2 Agent 模板库 (Agent Templates)
- **内置模板**：提供常见业务场景的 Agent 执行模板（如 B2B 登录验证、电商搜索加购）。
- **自定义模板沉淀**：允许将某次成功的 Agent 探索轨迹保存为 Schema 模板，供后续持续集成（CI）稳定复用。

### 2.3 任务调度与并发
- **并行执行策略**：支持多开 Magentic-UI 沙箱实例并发测试不同模块。
- **定时与触发器**：支持 Cron 表达式定时执行，或通过 Webhook 触发。

## 3. 智能执行与人机协同模块 (AaaS Execution & HIL)

这是平台的核心 AaaS 功能，由 LangGraph 和 Fara-7B 驱动。

### 3.1 实时可视化控制台 (Live Console)
- **画面推流**：在测试执行过程中，通过 WebSocket 实时推送 Magentic-UI 的沙箱浏览器画面。
- **状态树展示**：可视化展示当前 LangGraph 的状态机节点（初始化 -> 探索 -> 执行 -> 验证）。
- **思维链展示**：实时输出 Fara-7B 的推理过程（Thought / Action / Expected Result）。

### 3.2 人机协同阻断 (Human-in-the-Loop)
- **阻断策略配置**：用户可配置触发 HIL 的条件（如：遇到支付按钮、置信度 < 80%、页面出现 404 错误）。
- **协同审批工单**：当 Agent 触发阻断时，在控制台弹出提示工单，展示“当前截图”、“Agent意图”及“风险提示”。
- **人工干预操作**：
  - **同意**：允许 Agent 继续执行。
  - **拒绝并接管**：人工在可视化控制台上直接点击/输入，系统录制人工操作并反馈给 Agent，Agent 学习后继续后续步骤。
  - **终止**：结束测试并标记失败。

---

## 4. 测试资产与报告模块

### 4.1 智能测试报告生成
- **执行摘要**：展示测试结果（Pass/Fail）、总耗时、总步数、发现缺陷数。
- **执行轨迹回放**：以时间轴的形式展示每一步的“截图 -> 操作 -> 验证结果”，支持视频格式导出。
- **缺陷分析**：针对失败的节点，Fara-7B 自动生成缺陷原因分析（如“期望看到‘支付成功’，但页面弹出‘余额不足’提示”）。

### 4.2 资产管理 (Assets Management)
- **截图与录屏库**：分类管理测试过程中的多媒体资产。
- **DOM/UI 快照**：保存出错时的页面结构快照，辅助开发人员 Debug。

---

## 5. 开放 API 与系统集成模块 (Open API)

遵循 SDD 范式，系统所有能力均通过标准 API 暴露。

### 5.1 CI/CD 集成
- **插件支持**：提供 Jenkins、GitLab CI、GitHub Actions 插件。
- **状态回调**：任务完成后通过 Webhook 向第三方系统推送测试结果 Schema。

### 5.2 缺陷管理系统对接
- **一键建单**：测试失败时，自动将带有复现步骤、截图的报告推送到 Jira、Tapd 等项目管理工具。
