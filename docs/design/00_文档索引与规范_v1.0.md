# 黑盒自动化测试工具 - 文档索引与规范 (v1.0)

## 1. 文档体系（SDD 优先）

本项目采用 SDD（Schema-Driven Development）范式，核心约束：
- 所有跨模块数据交互必须先有 Schema，再有实现
- 文档中的关键对象（Task/Instance/Action/State/Ticket/Report）在 Schema 中具备唯一来源
- 设计文档与接口文档以“可落地实现”为验收标准

## 2. 目录与命名规范

- 目录：`docs/design/`
- 命名：`NN_文档名称_vX.Y.md`
- 版本：
  - `v1.0`：首版
  - `v1.1`：小幅增补，不破坏结构
  - `v2.0`：架构或核心对象变更，可能破坏兼容

## 4. 建议补充的工程化文档（常用于走向生产）

以下文档在 SaaS/AaaS 形态下通常必需或强烈建议（可按阶段补齐）：

### 4.1 产品与交付
- 产品需求文档（PRD）：范围、目标、里程碑、非目标
- 用户故事与验收标准（UAT）：按角色列出可验证的验收条款

### 4.2 SDD：Schema 与契约
- Schema 规范文档：字段约束、版本策略、兼容性规则
- 变更记录（Changelog）：按 Schema/API 版本出具

### 4.3 架构决策
- ADR（Architecture Decision Record）：关键技术选型与取舍记录

### 4.4 安全与合规
- 威胁模型与安全设计：沙箱隔离、敏感信息处理、审计策略
- 数据生命周期与存储策略：截图/录屏加密、保留期、销毁

### 4.5 运维与SRE
- 部署与运维手册（Runbook）：K8s 部署、扩缩容、故障处理
- 可观测性规范：指标/日志/追踪（Instance 维度链路）
- SLA/SLO：可用性、延迟、恢复时间

### 4.6 测试与质量
- 测试策略：单测/集成/端到端/回归基准
- 基准用例集：用于监控模型能力退化与推流稳定性
