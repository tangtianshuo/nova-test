# 黑盒自动化测试工具 - UI视觉设计规范 (v1.0)

## 1. 设计目标

本规范用于统一 SaaS 控制台的视觉语言与组件表现，保证：
- 信息密度高但不压迫（面向测试/运维工作流）
- 状态可视化清晰（运行中/阻塞/HIL/失败/成功）
- 实时性体验稳定（截图推流、日志流、状态机节点流转）

## 2. 品牌与基础视觉

### 2.1 色彩体系（语义优先）

#### 语义色
- `Primary`：主按钮、关键链接、选中态
- `Success`：成功、通过、已完成
- `Warning`：风险、需要关注、低置信度
- `Danger`：失败、阻断、高危
- `Info`：中性信息、提示

#### 中性色
- `BG`：页面背景
- `Surface`：卡片/面板
- `Border`：分割线
- `Text`：正文/次级/弱化

### 2.2 字体与排版

- 标题层级：`H1/H2/H3`，页面只保留一个 `H1`
- 正文：`Body`（默认）与 `BodyDense`（表格、日志列表）
- 数字/时间：统一使用等宽数字（tabular numbers）以利对齐

### 2.3 栅格与间距

- 页面左右留白：`24px`（桌面），`16px`（平板/移动）
- 基础间距单位：`4px` 的倍数（4/8/12/16/24/32）
- 信息密集区（表格、日志）：行高与间距采用 `dense` 变体

## 3. 组件规范（Design System）

### 3.1 Button

- `Primary`：主要动作（创建任务、启动执行、确认审批）
- `Secondary`：次要动作（保存草稿、返回）
- `Danger`：破坏性动作（终止执行、删除任务）
- 状态：`default/hover/active/disabled/loading`

### 3.2 Badge / Tag

- 用途：状态展示（任务状态、实例状态、风险等级、模型模式）
- 规则：单行不换行；超过宽度使用省略；支持 tooltip 展示全量

### 3.3 Table（任务列表/实例列表/工单列表）

- 列宽：优先给 `name/url/status/updatedAt` 分配固定或半固定宽度
- 行内操作：最多 3 个主入口，其余收纳到 `More` 菜单
- 筛选区：支持 `状态`、`时间范围`、`创建人`、`标签`、`环境` 组合过滤

### 3.4 Stepper / Timeline（执行轨迹）

- `Stepper`：展示 LangGraph 节点（Init/Explore/Execute/Verify/Report）
- `Timeline`：展示每一步的“截图/动作/验证结果/HIL 事件”
- 每一步必须明确：`StepNo`、`nodeName`、`耗时`、`结论`、`可回放`

### 3.5 Console（实时日志区）

- 支持分级：`INFO/WARN/ERROR/DEBUG`
- 默认折叠 `DEBUG`，可一键开启
- 支持关键字高亮与复制
- 无滚动条模式：默认展示最近 N 条，提供“上一页/下一页”分页与筛选
- 错误日志在顶部提供聚合摘要（最近 10 条）

### 3.6 Screenshot Viewer（实时画面/回放画面）

- 画面与信息双列：左侧截图，右侧动作与断言
- 支持标注层（可选）：展示 `target` 点击区域框与置信度
- 支持缩放、适配、全屏、下载

### 3.7 HIL Ticket Panel（人机协同工单面板）

- 必显字段：`原因`、`风险等级`、`计划动作`、`置信度`、`当前截图`
- 可操作：`批准`、`拒绝`、`修改动作`、`接管操作`、`终止实例`
- 高危默认：必须二次确认

## 4. 页面级视觉规范

### 4.1 信息架构（建议）

- 顶部：Workspace 切换、用户、通知
- 左侧导航：任务、执行实例、HIL 工单、资产、集成、计费、设置
- 主区：列表/详情/实时控制台

### 4.2 关键页面布局

#### 任务创建（Wizard）
- Step 1：目标与范围（URL、Objective、约束）
- Step 2：环境与账号（测试账号池、变量）
- Step 3：风险策略（HIL 触发条件、禁用域名、最大步数）
- Step 4：预览与提交

#### 实例执行详情（Live）
- 上：状态摘要卡（状态、耗时、步数、失败数、阻塞数）
- 中：截图推流 + 动作/断言
- 下：日志控制台 + Timeline（分页呈现，避免滚动条）

### 4.3 全屏无滚动条规范

- 页面必须适配 `100vh`，禁止出现浏览器级滚动条
- 面板内容禁止使用滚动条；超量信息采用“分页/折叠/抽屉/切换视图”呈现
- 列表默认单屏展示：任务/实例列表 5~8 行；日志 8~12 行；时间线 5~8 步
- 文本溢出统一省略（ellipsis）+ tooltip；JSON/长文本采用“截断预览 + 复制全文”
- 弹层/抽屉同样禁止滚动条：内容超过高度时，优先减少展示密度或切换分页

#### 报告详情
- 执行摘要 + 缺陷列表 + 轨迹回放

## 5. 可访问性与可用性

- 颜色不作为唯一信息源：状态必须配合文字/图标
- 键盘可达：主要操作支持 Tab 顺序与快捷键（启动、暂停、批准）
- 文本对比度满足常规可读性要求
- 重要操作（删除、终止、高危批准）二次确认

## 6. 视觉验收标准

- 状态一致性：列表、详情、Badge、Timeline 对同一状态的视觉表现一致
- 信息密度：任务列表单屏可展示足够字段且不拥挤
- 实时体验：截图更新时无明显布局抖动；日志与时间线分页切换无明显闪烁
