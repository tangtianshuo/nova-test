## 1. 范围

### 1.1 目标
- 以 SaaS 形态对外提供黑盒自动化测试能力
- 以 AaaS（Agent as a Service）形态提供“任务 -> 实例 -> 轨迹 -> 报告/模板”的闭环
- 以 SDD（Schema Driven Development）驱动研发：先定义 Schema/契约，再实现服务与前端

### 1.2 非目标（v1.0）
- 不追求 0 人工介入的全自动化；HIL 为默认能力
- 不覆盖移动端真机/桌面客户端（先聚焦 Web 浏览器沙箱）
- 不提供完整的商业计费闭环（v1.0 提供额度与用量统计，支付对接可延后到 P1）

## 2. 术语与对象模型

### 2.1 关键对象
- `Tenant/Workspace`：租户空间
- `User`：租户成员
- `Task`：测试任务定义（长期资产）
- `Instance`：一次执行实例（可暂停/恢复）
- `Step`：实例中的执行步骤（对应 LangGraph 节点流转与一次动作）
- `HIL Ticket`：人机协同阻断工单
- `Report`：结果汇总与可回放资产
- `Template`：由成功轨迹沉淀的可复用模板（Schema）

### 2.2 关键状态
- Instance：`PENDING`/`RUNNING`/`PAUSED_HIL`/`SUCCESS`/`FAILED`
- Ticket：`WAITING`/`APPROVED`/`REJECTED`/`MODIFIED`

## 3. 用户与权限（RBAC）需求

### FR-IAM-001（P0）租户隔离
- 描述：系统必须支持多租户隔离，任何租户只能访问自身数据与资源。
- 验收：
  - 使用同一账号在不同租户中切换，列表与详情数据完全隔离
  - API 访问携带 `tenant_id`，后端强制校验 token 所属租户

### FR-IAM-002（P0）角色与权限
- 描述：至少支持 `TenantAdmin`、`Architect`、`Tester`、`Viewer` 四类角色。
- 验收：
  - Viewer 无法创建/启动任务与实例
  - Tester 可创建任务、启动实例、处理 HIL 工单（按策略）
  - Architect 可维护模板与风险策略
  - TenantAdmin 可管理成员与额度

### FR-IAM-003（P1）成员邀请
- 描述：支持通过邮箱邀请成员加入租户。
- 验收：
  - 邀请链接/邮件可用，过期与撤销生效
  - 新用户接受邀请后自动加入租户并分配角色

## 4. 任务管理需求（Task）

### FR-TASK-001（P0）创建任务
- 描述：用户可通过 UI 或 API 创建 Task，至少包含 `name`、`target_url`、`natural_objective`、`constraints`。
- 验收：
  - 必填字段校验：URL 格式合法、objective 非空
  - constraints 至少包含 `max_steps`、`forbidden_domains` 的默认值或可选设置

### FR-TASK-002（P0）编辑与归档
- 描述：支持对任务进行编辑、归档（不影响历史 Instance/Report 的可追溯）。
- 验收：
  - 归档后任务不出现在默认列表，可在“归档”筛选中查看
  - 归档任务仍可查看历史实例与报告

### FR-TASK-003（P1）任务模板化
- 描述：支持从模板创建任务，或将任务保存为模板（仅限 Architect/Admin）。
- 验收：
  - 模板列表可检索、可复制
  - 从模板创建任务后可覆盖 URL、环境变量、风险策略

### FR-TASK-004（P1）定时/触发器
- 描述：支持定时运行或 webhook 触发运行（CI/CD 场景）。
- 验收：
  - 定时任务按 Cron 触发并产生 Instance
  - webhook 触发可携带参数覆盖某些运行时变量

## 5. AaaS 执行与状态机需求（Instance/Step）

### FR-AAS-001（P0）启动实例
- 描述：用户对某 Task 发起运行，系统创建 Instance，并分配沙箱浏览器环境。
- 验收：
  - Instance 状态从 `PENDING` -> `RUNNING`
  - 初始化步骤生成第一张截图并可在 UI 实时看到

### FR-AAS-002（P0）Observe-Think-Act 循环
- 描述：Instance 在 LangGraph 状态机驱动下循环：截图 -> 决策 -> 执行 -> 验证。
- 验收：
  - 每一步至少记录：截图、action_type、confidence、expected_result、验证结论
  - 达到 `max_steps` 或 action_type=`end` 后结束

### FR-AAS-003（P0）步骤轨迹可回放
- 描述：每个 Instance 必须有可查询的 Step 列表；报告页可按 Step 回放。
- 验收：
  - 通过 Step 编号可定位到对应截图与动作
  - UI 上点击 Timeline 任一步，画面与动作卡同步跳转

### FR-AAS-004（P0）失败与重试策略
- 描述：当验证失败或执行异常时，系统按策略重试或标记失败。
- 验收：
  - 失败会落库，Instance 状态变为 `FAILED`
  - 达到重试上限后不再无限循环

### FR-AAS-005（P1）断点续测
- 描述：基于 LangGraph checkpointer，在实例暂停/中断后可恢复。
- 验收：
  - 服务重启后，仍可从最新 checkpoint 继续
  - 恢复后 Step 编号连续、日志与截图链路完整

## 6. 人机协同（HIL Ticket）需求

### FR-HIL-001（P0）阻断策略
- 描述：当命中阻断规则（敏感操作、低置信度、异常循环等）时，实例进入 `PAUSED_HIL` 并创建 HIL 工单。
- 验收：
  - UI 在“工单”与“实例详情”均显示阻断原因与风险等级
  - 工单包含 planned_action、截图、置信度

### FR-HIL-002（P0）工单处理动作
- 描述：至少支持 `批准继续`、`拒绝`、`修改动作` 三种处理方式。
- 验收：
  - 批准后实例恢复执行
  - 修改动作后实例执行修改后的 action
  - 拒绝后按策略跳过或终止实例（默认终止）

### FR-HIL-003（P1）人工接管
- 描述：支持进入“人工接管模式”，人工直接操作沙箱页面，系统记录操作并回写给实例。
- 验收：
  - 接管期间 UI 有显著状态提示
  - 接管操作形成 Step，并在回放中可见

## 7. 实时推流与可观测性需求

### FR-RT-001（P0）截图推流
- 描述：实例运行时向前端推送实时截图更新。
- 验收：
  - 1 秒级别内可看到新截图（以实际网络为准）
  - 推流中断时有重连提示，保持最后一帧

### FR-RT-002（P0）Agent Thought 推送
- 描述：在每一步展示 thought/confidence/expected_result。
- 验收：
  - 右侧动作卡完整展示字段
  - 低置信度触发 warning 视觉表现（按颜色设计）

### FR-OBS-001（P1）实例级可观测性
- 描述：支持按 instance_id 查询关键指标与日志聚合。
- 验收：
  - 至少提供：总耗时、步数、HIL 次数、失败次数、重试次数

## 8. 报告与资产管理需求

### FR-REP-001（P0）报告生成
- 描述：实例结束后生成报告，包含摘要、缺陷列表、轨迹。
- 验收：
  - 报告可从任务/实例入口访问
  - 缺陷步骤可直接跳转到对应 Step

### FR-REP-002（P0）资产存储
- 描述：截图/录屏等大文件存储在对象存储（S3/MinIO），数据库保存 URL。
- 验收：
  - Step 记录 screenshotUrl
  - 访问 URL 受鉴权控制（预签名或网关代理）

### FR-REP-003（P1）导出
- 描述：支持导出报告为 `HTML/PDF/JSON`（JSON 供 CI 解析）。
- 验收：
  - JSON 导出包含 schema_version 与关键统计字段

## 9. 开放 API 与集成需求

### FR-API-001（P0）核心 API
- 描述：提供 Task 创建、Instance 触发、Instance 状态查询、HIL 恢复 API。
- 验收：
  - 与接口设计文档一致（路径、字段名、错误码）
  - 参数校验失败返回 `ERR_400`

### FR-API-002（P1）Webhook 回调
- 描述：实例结束后可通过 webhook 推送结果。
- 验收：
  - webhook payload 提供 pass/fail、缺陷数、报告链接

### FR-API-003（P2）缺陷系统对接
- 描述：支持 Jira/Tapd 一键建单。
- 验收：
  - 失败步骤截图与复现路径自动附加

## 10. 非功能需求（NFR）

### NFR-PERF-001（P0）单步延迟
- 目标：截图 -> 推理 -> 执行 的单步耗时 2-5 秒（取决于模型部署形态与网络）。

### NFR-SEC-001（P0）数据加密与生命周期
- 目标：截图/日志加密存储；按租户设置保留期与自动销毁。

### NFR-SEC-002（P0）沙箱隔离
- 目标：每个实例在隔离容器/Pod 内运行，最小权限，禁止访问内网敏感网段。

### NFR-REL-001（P1）容错与恢复
- 目标：服务重启后实例可从 checkpoint 恢复；推流断开可重连。

### NFR-COM-001（P1）审计
- 目标：关键动作（终止、批准高危、修改动作）必须记录操作者、时间与原因。
