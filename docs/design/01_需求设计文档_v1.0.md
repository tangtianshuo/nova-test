# 黑盒自动化测试工具 - 需求设计文档 (v1.0)

## 1. 引言

### 1.1 项目背景
基于前期《黑盒自动化测试工具技术实现可行性分析文档》，本项目旨在利用 **LangGraph**（Agent编排）、**Fara-7B**（端侧视觉操作代理）和 **Magentic-UI**（人机协作）等前沿AI技术，打造下一代智能化、视觉驱动的黑盒自动化测试平台。

### 1.2 核心理念
- **SaaS化 (Software as a Service)**：提供多租户、开箱即用的云端/私有化部署服务，采用按量计费或订阅制。
- **AaaS (Agent as a Service)**：将自动化测试能力抽象为独立的Agent服务，用户可通过API或UI调用具有“感知-思考-行动”能力的测试代理。
- **SDD (Schema-Driven Development)**：基于Schema先行的开发范式，统一数据结构、接口规范和Agent状态定义。

## 2. 目标用户与角色设定

### 2.1 目标用户群
- **QA/测试工程师**：无需编写脚本即可完成复杂业务流程的自动化回归测试。
- **研发工程师**：快速验证功能正确性，集成到CI/CD流水线。
- **产品经理/业务人员**：使用自然语言描述需求，直接生成验收测试（UAT）用例。

### 2.2 角色权限（RBAC）
| 角色 | 核心权限 |
| --- | --- |
| 平台管理员 | 系统配置、全局租户管理、资源监控、模型管理 |
| 租户管理员 | 组织架构管理、成员邀请、账单与计费查看、权限分配 |
| 测试架构师 | 测试环境配置、全局Agent参数调优、公共测试资产管理 |
| 测试执行者 | 创建测试任务、审批人机协同(HIL)节点、查看并导出报告 |

## 3. 业务需求说明

### 3.1 SaaS化基础需求
- **多租户隔离**：保证各企业/团队之间的数据、测试资产、操作日志严格逻辑隔离。
- **计费与额度管理**：基于“Agent运行时长”和“AI模型调用次数（Tokens/截屏数）”的计费模型。
- **开放平台 (Open API)**：允许企业将测试能力集成至自有的DevOps平台或质量管理系统。

### 3.2 AaaS (Agent as a Service) 核心需求
- **按需分配Agent**：用户提交任务后，系统动态调度资源，实例化基于 LangGraph 编排的 Fara-7B + Magentic-UI 测试Agent。
- **智能探索测试**：用户仅输入“自然语言目标”及“初始URL”，Agent可自主探索页面、识别元素并制定测试路径。
- **视觉断言验证**：基于Fara-7B模型对比操作前后的UI截图，完成无DOM依赖的视觉结果验证。
- **实时干预 (Human-in-the-Loop)**：在遇到高危操作（如支付、删除）、低置信度识别或异常阻断时，Agent能实时暂停并通过UI/WebSocket向测试人员发起协同请求。

### 3.3 核心业务流程
1. **任务定义**：输入目标地址、测试目标描述（自然语言）、约束条件。
2. **环境分配**：AaaS引擎分配无头浏览器（沙箱环境）并初始化测试会话。
3. **自主执行**：Agent在 LangGraph 状态机驱动下循环“观察(截图) -> 思考(Fara-7B决策) -> 动作(Magentic-UI执行) -> 验证”。
4. **人工协同**：触发阻断规则时，通过SaaS控制台发送审批请求。
5. **资产沉淀**：执行结束自动生成测试报告（含视频/截图轨迹、Bug列表），并将成功路径沉淀为标准Schema模板供后续复用。

## 4. 非功能需求

### 4.1 性能与可扩展性需求
- **弹性扩缩容**：AaaS执行引擎（尤其是GPU推理节点和浏览器沙箱节点）需支持Kubernetes HPA，应对高并发测试任务。
- **端到端延迟**：单个Agent操作步骤（截图->推理->执行）耗时需控制在 **2-5秒** 以内。

### 4.2 安全性与合规需求
- **数据安全**：所有租户的截图和日志必须加密存储（AES-256），生命周期到期后自动销毁。
- **沙箱隔离**：Magentic-UI的浏览器环境必须在隔离的Docker/Pod中运行，禁止访问内部生产网络。
- **隐私保护**：对于输入框内的敏感信息（如密码、身份证号），支持在UI渲染层进行模糊化处理，不输入到大模型。

---

## 5. SDD (Schema-Driven) 规范约定

为了保证系统的可维护性与模块解耦，核心业务需求均以Schema定义为准：
1. **Task Schema**：定义测试任务的输入输出标准格式。
2. **Agent State Schema**：基于 `TypedDict` 定义LangGraph运行时的全局状态。
3. **Action/Event Schema**：规范所有Agent操作事件和人机协同回调的数据结构。

*(具体Schema结构将在详细设计及接口设计文档中体现)*
