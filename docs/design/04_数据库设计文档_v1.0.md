# 黑盒自动化测试工具 - 数据库设计文档 (v1.0)

本平台采用关系型数据库 **PostgreSQL** 作为核心存储，结合 **Redis** 进行状态与队列管理，并使用 **S3/MinIO** 存储大文件（截图、录屏）。设计遵循 **SDD (Schema-Driven Development)** 范式，以下核心表结构采用 Prisma Schema 语法进行描述。

## 1. 实体关系图 (ERD 简述)

- `Tenant` (1) <---> (N) `User` (租户包含多个用户)
- `Tenant` (1) <---> (N) `TestTask` (租户包含多个测试任务)
- `TestTask` (1) <---> (N) `AgentInstance` (一个任务可以触发多次Agent实例执行)
- `AgentInstance` (1) <---> (N) `TestStep` (一个实例包含多个测试步骤/状态机节点)

## 2. 核心数据表设计 (Prisma Schema)

### 2.1 租户与用户管理 (SaaS 基础)

```prisma
// 租户表
model Tenant {
  id            String   @id @default(uuid())
  name          String   @db.VarChar(100)
  planType      String   @default("FREE") // 订阅套餐: FREE, PRO, ENTERPRISE
  balance       Float    @default(0.0)    // 账户余额/额度
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  users         User[]
  tasks         TestTask[]
}

// 用户表
model User {
  id            String   @id @default(uuid())
  tenantId      String
  email         String   @unique @db.VarChar(255)
  passwordHash  String   @db.VarChar(255)
  role          String   @default("TESTER") // ADMIN, ARCHITECT, TESTER
  createdAt     DateTime @default(now())
  
  tenant        Tenant   @relation(fields: [tenantId], references: [id])
}
```

### 2.2 任务与 Agent 实例 (AaaS 核心)

```prisma
// 测试任务定义表
model TestTask {
  id               String   @id @default(uuid())
  tenantId         String
  name             String   @db.VarChar(255)
  targetUrl        String   @db.Text
  naturalObjective String   @db.Text         // 自然语言测试目标
  constraints      Json?                     // 约束条件 (Schema JSON)
  status           String   @default("ACTIVE") // ACTIVE, ARCHIVED
  createdAt        DateTime @default(now())
  updatedAt        DateTime @updatedAt

  tenant           Tenant          @relation(fields: [tenantId], references: [id])
  instances        AgentInstance[]
}

// Agent 实例执行表 (每次运行产生一个实例)
model AgentInstance {
  id               String   @id @default(uuid())
  taskId           String
  status           String   @default("PENDING") // PENDING, RUNNING, PAUSED_HIL, SUCCESS, FAILED
  langgraphState   Json?                     // LangGraph 状态机 Checkpoint 快照
  startedAt        DateTime?
  completedAt      DateTime?
  
  task             TestTask   @relation(fields: [taskId], references: [id])
  steps            TestStep[]
}
```

### 2.3 测试执行轨迹 (执行数据)

```prisma
// Agent 状态机执行步骤表
model TestStep {
  id               String   @id @default(uuid())
  instanceId       String
  stepNumber       Int
  nodeName         String   @db.VarChar(50)  // 状态机节点: explore, execute, verify, hil
  
  // 视觉与上下文数据
  screenshotUrl    String?  @db.Text         // 存储于 MinIO 的对象 URL
  
  // Fara-7B 决策数据
  thought          String?  @db.Text
  actionType       String?  @db.VarChar(50)  // click, type, scroll, wait
  actionTarget     Json?                     // 操作目标坐标/选择器
  
  // 验证结果
  isSuccess        Boolean?
  verificationMsg  String?  @db.Text
  
  createdAt        DateTime @default(now())

  instance         AgentInstance @relation(fields: [instanceId], references: [id])
}
```

### 2.4 人机协同 (HIL) 记录表

```prisma
// 人工干预工单表
model HilTicket {
  id               String   @id @default(uuid())
  instanceId       String   @unique          // 关联到被阻塞的 Agent 实例
  reason           String   @db.Text         // 阻断原因 (如：检测到支付按钮)
  riskLevel        String   @default("MEDIUM") // HIGH, MEDIUM, LOW
  status           String   @default("WAITING") // WAITING, APPROVED, REJECTED, MODIFIED
  
  decisionPayload  Json?                     // 人工决策结果的 Schema 数据
  resolvedBy       String?                   // 处理人 User ID
  resolvedAt       DateTime?
  createdAt        DateTime @default(now())
}
```

## 3. 缓存与队列设计 (Redis)

- **任务队列 (`queue:agent_tasks`)**：采用 List 或 Redis Streams，保存待执行的 `AgentInstance` ID。Worker 节点（Magentic-UI 容器）监听并消费。
- **WebSocket 状态发布 (`pubsub:agent_stream:<instance_id>`)**：利用 Redis Pub/Sub 机制，Agent 运行时不断将最新状态（截图 Base64 / 节点信息）发布至该频道，SaaS 平台的 WebSocket 网关订阅并推送到前端 UI。
- **会话锁定 (`lock:hil:<instance_id>`)**：防止多个人员同时处理同一个 HIL 审批工单。
