# 黑盒自动化测试工具 - 详细设计文档 (v1.0)

本详细设计聚焦于 AaaS (Agent as a Service) 引擎的核心执行逻辑，即基于 **LangGraph**、**Fara-7B** 和 **Magentic-UI** 的整合架构。设计严格遵循 **SDD (Schema-Driven Development)**，通过明确的数据模式定义系统行为。

## 1. SDD: Agent State Schema 设计

LangGraph 是基于状态机（State Machine）的工作流框架。所有节点之间传递的唯一信源是全局 State，我们使用 Python `TypedDict` 进行 Schema 定义。

### 1.1 全局状态 (AgentState)

```python
from typing import TypedDict, Annotated, List, Dict, Any, Optional
import operator

class AgentState(TypedDict):
    """LangGraph 全局运行状态 Schema"""
    instance_id: str             # Agent 实例ID
    target_url: str              # 目标测试地址
    test_objective: str          # 自然语言描述的测试目标
    
    status: str                  # 状态: running, paused_hil, completed, failed
    current_step: int            # 步数计数器
    
    # 视觉上下文
    screenshots: Annotated[List[str], operator.add] # 存储于MinIO的截图URL列表
    current_screenshot_b64: str  # 当前最新截图(用于实时推理和WS推流)
    
    # 操作与决策序列
    action_history: Annotated[List[Dict], operator.add] # 历史操作轨迹
    planned_action: Optional[Dict]                      # Fara-7B当前计划执行的动作
    
    # HIL 上下文
    hil_ticket_id: Optional[str] # 关联的审批工单ID
    hil_decision: Optional[Dict] # 人工干预返回的决策结果
```

### 1.2 操作行为 Schema (ActionSchema)

这是 Fara-7B 输出以及 Magentic-UI 接收的标准动作格式：

```python
from pydantic import BaseModel, Field

class ActionTarget(BaseModel):
    x: Optional[float] = Field(None, description="屏幕X坐标")
    y: Optional[float] = Field(None, description="屏幕Y坐标")
    selector: Optional[str] = Field(None, description="备用DOM选择器")

class AgentAction(BaseModel):
    thought: str = Field(..., description="Fara-7B的思维链推理过程")
    action_type: str = Field(..., description="操作类型: click, type, scroll, wait, end")
    target: ActionTarget
    params: Dict[str, Any] = Field(default_factory=dict, description="操作参数，如输入文本")
    confidence: float = Field(..., description="操作置信度 0.0-1.0")
    expected_result: str = Field(..., description="预期操作后的页面状态描述")
```

## 2. LangGraph 状态机编排设计

我们定义一个有向循环图 (Cyclic Graph) 来驱动 Agent。

### 2.1 节点定义 (Nodes)

1. **`node_init`**: 初始化 Magentic-UI 沙箱，打开 `target_url`，捕获第一张截图并写入 `current_screenshot_b64`。
2. **`node_explore`**: 调用 Fara-7B 模型。传入 `current_screenshot_b64`、`test_objective` 和 `action_history`，通过 Prompt 让模型输出符合 `AgentAction` Schema 的 JSON。
3. **`node_check_hil`**: 拦截器节点。检查 `planned_action` 的置信度（如 < 0.8）或是否包含敏感操作（如支付）。如果触发规则，将状态设为 `paused_hil` 并挂起图执行（LangGraph Checkpointer 保存状态）。
4. **`node_execute`**: Magentic-UI 接收 `planned_action`，在 Playwright 浏览器中执行实际物理操作（点击、输入等），并捕获操作后的新截图。
5. **`node_verify`**: 将操作前截图、操作后截图和 `expected_result` 送入 Fara-7B 进行结果比对验证。

### 2.2 边与条件路由 (Edges & Conditional Routing)

```python
# 核心路由逻辑伪代码
workflow.add_edge("node_init", "node_explore")
workflow.add_edge("node_explore", "node_check_hil")

def route_from_hil(state: AgentState):
    if state["status"] == "paused_hil":
        return END # 挂起，等待外部恢复
    return "node_execute"

workflow.add_conditional_edges("node_check_hil", route_from_hil)
workflow.add_edge("node_execute", "node_verify")

def route_after_verify(state: AgentState):
    if state["current_step"] > MAX_STEPS:
        return END
    if state["action_history"][-1]["action_type"] == "end":
        return END
    return "node_explore" # 循环

workflow.add_conditional_edges("node_verify", route_after_verify)
```

## 3. 人机协同 (Human-in-the-Loop) 的唤醒与接管设计

基于 LangGraph 的 Checkpointer（如 PostgreSQL Saver）机制实现异步的 HIL。

### 3.1 挂起 (Suspend)
当 `node_check_hil` 发现风险时，抛出特定异常或通过条件路由将图执行结束（End），但此时 State 保存在 DB 中，线程 ID 为 `instance_id`。系统通过 Redis 发送一个 HIL 事件给 SaaS 控制面。

### 3.2 恢复 (Resume)
1. 租户管理员在 Web UI 收到工单，点击“同意继续”或“修改操作（如更正点击坐标）”。
2. 前端调用 API `/api/v1/agents/{instance_id}/resume`。
3. API 更新 DB 中的 `AgentState`（注入 `hil_decision`），然后调用 `graph.invoke(None, config={"configurable": {"thread_id": instance_id}})` 唤醒图。
4. 状态机在上次挂起的地方恢复，并进入 `node_execute` 执行人工修正后的操作。

## 4. Fara-7B 模型集成设计

Fara-7B 运行在独立的 GPU 节点上（如通过 vLLM 或 TGI 部署为兼容 OpenAI 接口的 API 服务）。

### 4.1 视觉感知与提示词工程 (Prompt Engineering)

为了让 Fara-7B 输出稳定的 SDD JSON，我们采用以下 Prompt 结构：
- **System Prompt**: 扮演专业测试Agent，仅输出符合 ActionSchema 的 JSON。
- **User Prompt**: 
  - `Objective`: 用户设定的测试目标
  - `History`: 之前步骤的摘要列表
  - `Image`: 当前截图 (Base64)
- **Logit Processors**: 在推理端启用 JSON Mode，强制约束 Fara-7B 生成合规的 JSON 结构，避免解析失败。
