# 黑盒自动化测试工具 - UE/UX交互设计文档 (v1.0)

## 1. 交互目标与原则

### 1.1 交互目标
- 让测试任务“可配置、可观测、可接管、可复用”
- 将 Agent 的不确定性显性化（置信度、风险、下一步意图）
- 用最少的人工操作完成关键决策（Human-in-the-Loop）

### 1.2 设计原则
- 状态优先：所有页面以“任务/实例/工单”的状态为第一信息
- 实时不打扰：实时推流默认稳定呈现，异常才强提醒
- 低学习成本：以测试工程师常用心智模型组织信息（列表 -> 详情 -> 回放）
- 全屏无滚动条：页面适配 `100vh`，列表/日志/时间线采用分页或折叠呈现

## 2. 用户角色与核心任务

### 2.1 角色
- `租户管理员`：成员/权限、计费、额度、集成
- `测试架构师`：模板、环境策略、HIL 规则、公共资产
- `测试执行者`：创建任务、运行实例、处理工单、产出报告

### 2.2 核心任务（Top Tasks）
- 创建任务并启动一次执行
- 观察执行过程，必要时人工接管/批准
- 查看失败原因并导出报告/一键建单
- 将成功轨迹沉淀为模板，放入 CI 定时执行

## 3. 信息架构（IA）与导航

### 3.1 一级导航（建议）
- 仪表盘
- 任务
- 执行实例
- HIL 工单
- 报告
- 资产库
- 集成（Webhook/CI/Jira）
- 计费与额度
- 设置（环境、账号池、模型、风险策略）

### 3.2 关键对象层级
- `Task`：长期存在的测试定义
- `Instance`：一次运行产生的执行实体（可暂停/恢复）
- `Ticket(HIL)`：一次阻断产生的协作工单
- `Report`：一次或多次实例的结果汇总（可导出、可建单）

## 4. 关键页面与交互细节

### 4.1 登录与 Workspace 选择

- 登录成功后进入 Workspace 选择
- Workspace 为空时引导创建并提供“从模板快速开始”入口

### 4.2 任务列表页

#### 核心交互
- 默认按 `updatedAt` 倒序
- 快捷过滤：`全部/进行中/失败/需要处理(HIL)`
- 行操作：`启动实例`、`编辑`、`更多(归档/复制/删除)`

#### 空态
- 空态提供 3 个入口：`新建任务`、`从模板创建`、`导入 YAML/JSON Schema`

### 4.3 任务创建向导（Wizard）

#### Step 1：目标与范围
- 输入：`target_url`、`natural_objective`
- 可选：`forbidden_domains`、`max_steps`
- 即时校验：URL 格式、目标文字长度

#### Step 2：环境与账号
- 选择环境：`dev/stage/prod(受限)`
- 账号池：支持绑定“变量名”并在运行时注入
- 安全规则：敏感字段在 UI 显示掩码，不进入模型上下文

#### Step 3：风险与 HIL 策略
- 预设策略：`保守/均衡/激进`
- 可配置阈值：`confidence_threshold`
- 敏感操作清单：支付、删除、发送邮件、下载文件

#### Step 4：预览与提交
- 展示结构化预览（Schema）
- 提供 `保存为模板` 选项

### 4.4 实例执行详情页（Live Console）

#### 页面布局
- 左侧：截图/回放画面
- 右侧：当前动作卡（thought、action、confidence、expected_result）
- 右侧下方：日志 Console + Timeline（分页呈现，避免滚动条）

#### 实时交互
- 截图推流：默认“跟随最新”，用户手动翻历史后进入“暂停跟随”状态
- Timeline：点击某一步，画面与动作卡同步跳转
- 日志：默认展示最近 N 条，支持“上一页/下一页”与“仅看错误”

#### 关键控制
- `暂停/继续`：对实例的软控制（不销毁沙箱）
- `终止`：破坏性操作，二次确认
- `导出现场`：打包导出当前截图、日志、状态快照

### 4.5 HIL 工单处理页

#### 工单呈现
- 必须明确：为何阻断、风险等级、计划动作、模型置信度、现场截图

#### 处理动作
- `批准继续`：继续执行计划动作
- `拒绝`：跳过或终止（策略可选）
- `修改动作`：编辑坐标/输入内容/替代策略
- `人工接管`：进入“接管模式”，允许人工在画面上直接操作，系统记录操作并回写给 Agent

#### 接管模式交互
- 顶部显著提示“人工接管中”
- 提供“回放刚才人工操作”的一键查看
- 退出接管：选择“交还给 Agent”或“终止实例”

### 4.6 报告页

#### 结构
- 执行摘要：通过/失败、耗时、步数、缺陷数、HIL 次数
- 缺陷列表：可按严重度、节点、页面过滤
- 轨迹回放：与 Timeline 同构

#### 集成动作
- `一键建单`：把失败步骤、截图、复现路径、环境信息推送到 Jira/Tapd
- `导出`：PDF/HTML/JSON（供 CI 解析）

## 5. 提示、通知与异常体验

### 5.1 通知分级
- 低优先：正常状态变化（进入下一节点）不弹窗，仅在顶部消息中心记录
- 中优先：实例失败、重试次数过多，给到 toast + 消息中心
- 高优先：HIL 阻断、额度不足、权限不足，必须弹出阻断型对话框

### 5.2 常见异常与兜底
- 推流中断：画面区展示“重连中”，保留最后一帧并提示用户刷新或稍后重试
- 模型解析失败：提示“本步输出不符合 Schema”，自动进入重试/人工介入
- 超时：提供“继续等待/重试/终止”

## 6. 可用性验收标准

- 新手路径：首次创建任务到启动实例 <= 3 分钟
- HIL 处理：从收到工单到恢复执行 <= 30 秒完成一次“批准”
- 回放定位：从报告页定位到失败步骤的现场截图 <= 3 次点击
