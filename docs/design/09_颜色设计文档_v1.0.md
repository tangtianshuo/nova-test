# 黑盒自动化测试工具 - 颜色设计文档 (v1.0)

## 1. 目标与适用范围

本颜色体系用于 SaaS 控制台所有界面，包括：任务列表、实例执行控制台、HIL 工单、报告与资产库。设计目标是：
- 以“语义色”表达状态与风险，减少误判
- 支持高信息密度界面与长时间盯盘（可选深色模式）
- 保证在截图推流、日志高亮、图表分析等场景下的可读性

## 2. 颜色模型与 Token 规范（SDD 思路）

### 2.1 Token 分层
- `Base Tokens`：基础色板（灰阶、主色、功能色）
- `Semantic Tokens`：语义映射（Success/Warning/Danger/Info/Primary）
- `Component Tokens`：组件级颜色（按钮、表格、Badge、Tag、Toast）

### 2.2 命名规则
- 基础色：`color.<family>.<step>` 例如 `color.gray.100`
- 语义色：`semantic.<name>.<role>` 例如 `semantic.success.fg`
- 组件色：`component.<component>.<state>.<role>`

## 3. 基础色板（建议值）

### 3.1 灰阶（界面骨架）

| Token | Hex | 用途 |
| --- | --- | --- |
| `color.gray.0` | `#FFFFFF` | 纯白 |
| `color.gray.50` | `#F7F8FA` | 页面背景（浅） |
| `color.gray.100` | `#EEF0F3` | Surface 背景、弱分割 |
| `color.gray.200` | `#DDE1E6` | Border |
| `color.gray.400` | `#8A93A3` | 次级文字 |
| `color.gray.700` | `#2E3440` | 正文文字 |
| `color.gray.900` | `#121826` | 标题/强调文字 |

### 3.2 主色（品牌与主操作）

| Token | Hex | 用途 |
| --- | --- | --- |
| `color.primary.500` | `#3B82F6` | 主按钮、选中态 |
| `color.primary.600` | `#2563EB` | 主按钮 hover |
| `color.primary.50` | `#EFF6FF` | 主色浅底（Tag/背景） |

### 3.3 功能色（状态与风险）

| 语义 | Token | Hex | 用途 |
| --- | --- | --- | --- |
| 成功 | `color.success.500` | `#22C55E` | Pass、成功 |
| 警告 | `color.warning.500` | `#F59E0B` | 低置信度、需要关注 |
| 危险 | `color.danger.500` | `#EF4444` | Fail、阻断、高危 |
| 信息 | `color.info.500` | `#06B6D4` | 中性提示、运行中 |

## 4. 语义映射（Semantic Tokens）

### 4.1 全局语义

| 语义 | `bg` | `fg` | `border` | `mutedBg` |
| --- | --- | --- | --- | --- |
| `semantic.primary` | `color.primary.50` | `color.primary.600` | `color.primary.500` | `#DDEBFF` |
| `semantic.success` | `#ECFDF3` | `#15803D` | `color.success.500` | `#D1FAE5` |
| `semantic.warning` | `#FFFBEB` | `#B45309` | `color.warning.500` | `#FEF3C7` |
| `semantic.danger` | `#FEF2F2` | `#B91C1C` | `color.danger.500` | `#FEE2E2` |
| `semantic.info` | `#ECFEFF` | `#0E7490` | `color.info.500` | `#CFFAFE` |

### 4.2 状态到颜色的统一映射

#### Task/Instance 状态
- `PENDING`：`semantic.info`
- `RUNNING`：`semantic.info`
- `PAUSED_HIL`：`semantic.warning`
- `SUCCESS`：`semantic.success`
- `FAILED`：`semantic.danger`

#### 风险等级
- `LOW`：`semantic.info`
- `MEDIUM`：`semantic.warning`
- `HIGH`：`semantic.danger`

## 5. 组件用色规则

### 5.1 Badge/Tag

- 默认使用 `semantic.*.mutedBg` 作为背景，`semantic.*.fg` 作为文字
- 边框使用 `semantic.*.border` 或弱化 30%

### 5.2 Button

- `Primary`：背景 `color.primary.500`，文字 `color.gray.0`，hover `color.primary.600`
- `Danger`：背景 `color.danger.500`，文字 `color.gray.0`
- `Secondary`：背景 `color.gray.100`，边框 `color.gray.200`

### 5.3 Table

- 表头：`color.gray.50`
- 行 hover：`color.gray.100`
- 选中行：`color.primary.50`（避免与状态色冲突）

### 5.4 Log Console（高亮建议）

- `INFO`：文字 `color.gray.700`
- `WARN`：文字 `#92400E`，背景 `#FFFBEB`
- `ERROR`：文字 `#991B1B`，背景 `#FEF2F2`
- `DEBUG`：文字 `color.gray.400`

## 6. 深色模式（可选）

### 6.1 原则
- 深色模式优先保证长时间阅读舒适度，降低高亮饱和度
- 语义色保持“可辨识一致”，但整体亮度下调

### 6.2 建议映射
- `bg`：`#0B1220`
- `surface`：`#0F172A`
- `border`：`#1F2A44`
- `text`：`#E5E7EB`
- `textMuted`：`#9CA3AF`

## 7. 图表与数据可视化用色

- 颜色数量 <= 8，避免彩虹图
- 默认序列色板建议：
  - `#3B82F6`、`#22C55E`、`#F59E0B`、`#EF4444`、`#06B6D4`、`#A855F7`、`#64748B`、`#F97316`
- 成功/失败趋势图优先使用 `success/danger` 语义色

## 8. 可读性与对比度约束

- 状态色必须同时配合文字或图标表达
- 在截图标注层（bounding box）中，框线颜色需有 2px 描边或阴影以应对复杂背景
- 重要 CTA 不允许使用低对比度组合（如浅灰文字 + 白底）
