## 1. 执行摘要

### 1.1 项目背景

随着软件系统复杂度持续提升，传统黑盒测试面临测试用例设计成本高、维护困难、覆盖不全等挑战。基于大语言模型（LLM）和视觉语言模型（VLM）的智能代理（Agent）技术为自动化测试带来了范式变革机遇。

### 1.2 目标

评估基于 **LangGraph**（Agent编排框架）、**Fara-7B**（微软端侧计算机操作代理模型）和 **Magentic-UI**（微软人机协作自动化平台）构建黑盒自动化测试工具的技术可行性。

### 1.3 核心结论

**该技术方案具备高度可行性**，三者均为微软生态或兼容生态的开源组件，在架构设计、功能互补性和技术成熟度方面形成了天然的协同优势。预计可实现：

- 无需编写测试脚本的智能探索测试
- 基于视觉感知的端到端系统验证
- 人机协同的测试流程管控
- 端侧部署的低延迟执行

### 1.4 关键指标预览

| 指标 | 预期值 |
|------|--------|
| 元素识别准确率 | >85% |
| 测试用例生成速度 | 5-10倍于人工 |
| 端到端操作延迟 | 2-5秒/操作 |
| 人工介入率 | 10-20% |

## 2. 技术组件深度解析

### 2.1 LangGraph：Agent工作流编排引擎

LangGraph 是 LangChain 团队推出的基于**状态机（State Machine）**的智能体工作流设计框架，专为复杂、长期运行的 Agent 任务设计。

#### 2.1.1 核心特性

| 特性 | 说明 | 测试场景价值 |
|------|------|-------------|
| **显式状态管理** | 通过 TypedDict 定义全局状态 Schema，每个节点接收状态并输出更新 | 测试流程可预测、可调试 |
| **持久化与容错** | Checkpointer 机制支持断点续传、时间旅行调试 | 长流程测试可恢复、可审计 |
| **循环与分支** | 原生支持条件跳转、循环执行和回退重试 | 复杂测试逻辑灵活编排 |
| **可视化调试** | 提供图结构可视化工具 | 便于理解测试执行路径 |

#### 2.1.2 在黑盒测试中的定位

作为**测试流程的编排层**，负责：

1. **状态机管理**：维护测试用例的全生命周期状态
   - 初始化 → 页面探索 → 操作执行 → 结果验证 → 报告生成

2. **多Agent协调**：调度测试规划Agent、执行Agent、验证Agent协作

3. **异常恢复**：实现断点续测和智能回退机制

4. **人机介入点管理**：在关键节点暂停等待人工确认

### 2.2 Fara-7B：端侧视觉操作代理模型

Fara-7B 是微软于 2025年11月发布的**首个专为计算机操作设计的Agentic小语言模型（SLM）**，属于 Computer Use Agent (CUA) 类别。

#### 2.2.1 技术规格

| 属性 | 详情 |
|------|------|
| 参数规模 | 70亿参数 |
| 基础架构 | 基于 Qwen2.5-VL-7B 构建 |
| 上下文窗口 | 128K tokens |
| 输入模态 | 屏幕截图 + 历史操作记录 + 用户指令 |
| 输出模态 | 推理链（CoT）+ 工具调用（点击、输入、滚动等） |
| 训练数据 | 145K条合成轨迹（FaraGen生成） |
| 部署方式 | 本地端侧（支持NPU加速）或云端API |

#### 2.2.2 核心优势

**1. 纯视觉感知能力**
- 直接分析屏幕截图像素，无需依赖无障碍树（Accessibility Trees）或DOM结构
- 更接近人类真实交互方式，跨平台兼容性强

**2. 端侧部署优势**
- 7B参数规模支持在本地设备运行（包括Copilot+ PC的NPU）
- 降低网络延迟，保护数据隐私

**3. 观察-思考-行动循环**
- 通过"observe-think-act"闭环执行任务
- 具备自主决策和错误恢复能力

**4. 安全机制**
- 在关键操作点自动暂停等待用户确认
- 降低自动化测试的风险

#### 2.2.3 在黑盒测试中的定位

作为**测试执行引擎**，负责：

1. **视觉感知**：自动识别被测系统的UI元素（按钮、输入框、链接等）
2. **决策生成**：基于当前状态和测试目标生成下一步操作
3. **结果验证**：对比操作前后截图，验证预期结果

### 2.3 Magentic-UI：人机协作自动化平台

Magentic-UI 是微软于2025年5月开源的**以人为中心的网页Agent系统**，基于 Magentic-One 多智能体系统和 AutoGen 框架构建。

#### 2.3.1 架构组成

采用**五Agent协作架构**：

| Agent角色 | 职责 | 测试场景应用 |
|-----------|------|-------------|
| **Orchestrator（协调器）** | 制定计划、协调其他Agent、监控进度 | 测试任务分解与调度 |
| **WebSurfer（网页浏览代理）** | 执行浏览器操作（点击、输入、滚动） | 具体的页面交互执行 |
| **Coder（编程代理）** | 编写和执行Python/Shell代码 | 测试数据生成与处理 |
| **FileSurfer（文件处理代理）** | 处理文件转换和分析 | 测试报告生成 |
| **UserProxy（用户代理）** | 代表用户参与协作流程 | 人工介入接口 |

#### 2.3.2 核心功能

| 功能 | 说明 | 价值 |
|------|------|------|
| **协同规划** | 用户与AI共同制定、编辑任务执行计划 | 测试策略透明可控 |
| **实时可视化** | 提供操作过程的实时截图或视频流展示 | 测试过程可观察 |
| **操作审批** | 敏感操作需人工确认后方可执行 | 风险可控 |
| **计划学习** | 自动保存成功的工作流作为模板复用 | 测试资产积累 |
| **并行执行** | 支持同时运行多个独立任务 | 提升测试效率 |

#### 2.3.3 在黑盒测试中的定位

作为**测试执行环境和人机交互层**，负责：

1. **浏览器自动化**：提供基于Playwright的浏览器控制能力
2. **人机协同**：实现测试过程的透明化和人工介入机制
3. **资产管理**：管理测试截图、日志、代码片段
4. **模板复用**：支持测试用例的复用和标准化

### 3.2 数据流设计

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  测试需求输入  │────►│  LangGraph  │────►│  状态初始化  │────►│ 启动沙箱环境  │
│  (自然语言)   │     │   编排器     │     │             │     │(Magentic-UI)│
└─────────────┘     └─────────────┘     └─────────────┘     └──────┬──────┘
                                                                     │
                                                                     ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  生成测试报告  │◄────│  汇总执行轨迹  │◄────│  验证结果    │◄────│  捕获页面截图  │
│  (结构化输出) │     │  (状态聚合)   │     │  (Fara-7B)  │     │  (浏览器)     │
└─────────────┘     └─────────────┘     └──────┬──────┘     └─────────────┘
                                               │
                                               ▼
                                        ┌─────────────┐
                                        │  决策下一步   │
                                        │  (Fara-7B)   │
                                        └──────┬──────┘
                                               │
                                               ▼
                                        ┌─────────────┐
                                        │  执行操作    │
                                        │(Magentic-UI)│
                                        └─────────────┘
```

**详细流程说明：**

1. **测试初始化**
   - 用户输入测试需求（自然语言描述）
   - LangGraph 加载测试配置，初始化全局状态（目标URL、测试范围、约束条件）
   - Magentic-UI 启动沙盒浏览器实例

2. **页面探索**
   - Fara-7B 接收当前页面截图
   - 分析可交互元素（按钮、输入框、链接等）
   - 生成探索策略（优先测试路径）

3. **操作执行**
   - LangGraph 将决策传递给 Magentic-UI
   - WebSurfer 在沙盒浏览器中执行具体操作（点击、输入、滚动）
   - 捕获操作后页面状态

4. **状态验证**
   - Fara-7B 对比操作前后截图
   - 验证预期结果是否达成
   - 识别异常状态（错误提示、页面崩溃等）

5. **流程控制**
   - LangGraph 根据验证结果决定：
     - 成功 → 继续下一步探索
     - 失败 → 标记缺陷，尝试替代路径
     - 异常 → 触发回退或人工介入

6. **报告生成**
   - 聚合执行日志、截图、操作轨迹
   - 生成结构化测试报告（缺陷列表、覆盖分析、执行统计）

## 4. 可行性分析

### 4.1 技术兼容性分析

| 维度 | 评估等级 | 详细说明 |
|---------|------|----------|
| **生态兼容性** | ⭐⭐⭐⭐⭐ 优秀 | Fara-7B和Magentic-UI均为微软研究院出品，设计理念一致，共享AutoGen框架基础 |
| **接口对接** | ⭐⭐⭐⭐☆ 良好 | Fara-7B输出标准工具调用格式，Magentic-UI基于AutoGen框架，均可通过Python SDK与LangGraph集成 |
| **数据格式** | ⭐⭐⭐⭐⭐ 优秀 | 三者均支持Python生态，状态数据可通过Pydantic/TypedDict无缝传递，截图数据支持PIL/OpenCV格式 |
| **部署模式** | ⭐⭐⭐☆☆ 需注意 | Fara-7B支持端侧部署，Magentic-UI需Docker容器，需统一资源调度策略 |
| **版本兼容性** | ⭐⭐⭐⭐☆ 良好 | 均处于活跃开发期，需建立版本锁定机制，避免破坏性更新 |

**兼容性结论：** 技术栈间无明显兼容性障碍，主要挑战在于部署环境的统一配置。

### 4.2 功能互补性矩阵

| 功能需求 | LangGraph | Fara-7B | Magentic-UI | 协作方式说明 |
|----------|-----------|---------|-------------|-------------|
| **测试流程编排** | 主导 | 参与 | 参与 | LangGraph定义状态机，协调两者执行时序 |
| **视觉感知与决策** | 不涉及 | 主导 | 辅助 | Fara-7B分析截图，输出结构化决策 |
| **浏览器控制** | 不涉及 | 辅助 | 主导 | Magentic-UI执行具体操作，返回执行结果 |
| **人机协同** | 支持 | 不涉及 | 主导 | Magentic-UI提供UI，LangGraph管理中断点 |
| **异常恢复** | 主导 | 参与 | 参与 | LangGraph决策重试策略，两者配合执行 |
| **测试资产管理** | 不涉及 | 不涉及 | 主导 | FileSurfer管理截图、日志、代码片段 |
| **状态持久化** | 主导 | 不涉及 | 辅助 | LangGraph Checkpointer机制，支持断点续传 |
| **并行测试执行** | 支持 | 不涉及 | 主导 | Magentic-UI并行任务能力，LangGraph调度多实例 |

**互补性结论：** 三者形成功能闭环，无重叠冗余，覆盖黑盒测试全流程。

### 4.3 技术风险评估

| 风险项 | 等级 | 影响描述 | 缓解措施 | 残余风险 |
|--------|------|---------|----------|---------|
| **Fara-7B幻觉导致误操作** | 中 | 模型可能错误识别元素，导致点击错误位置 | 1. 利用Magentic-UI审批机制，关键操作人工确认<br>2. 设置置信度阈值，低置信度时暂停<br>3. 添加操作前元素位置二次验证 | 低 |

| **长流程状态管理复杂** | 中 | 多步骤测试场景状态爆炸，恢复困难 | 1. 利用LangGraph Checkpointer机制<br>2. 定期状态快照保存<br>3. 子流程模块化，降低复杂度 | 低 |

| **多Agent通信开销** | 低 | 三者间频繁数据传输引入延迟 | 1. 共享内存传递截图数据<br>2. 批量处理减少通信次数<br>3. 本地部署降低网络延迟 | 极低 |
| **安全沙箱逃逸** | 低 | 自动化操作可能突破沙箱限制 | 1. 使用Docker安全隔离<br>2. 限制浏览器权限（无文件系统访问）<br>3. 网络隔离，禁止访问内网资源 | 极低 |
| **依赖项更新冲突** | 低 | 三者为独立项目，版本更新可能引入不兼容 | 1. 建立版本锁定文件（requirements.lock）<br>2. CI/CD中自动化兼容性测试<br>3. 社区版本跟踪机制 | 极低 |

**风险评估结论：** 无高风险项，中风险项均有成熟缓解措施，整体风险可控。

#### 5.1.2 LangGraph状态机实现

```python
# test_framework/graph/test_flow.py
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.memory import MemorySaver
from test_framework.core.state import TestState
from test_framework.agents.fara_agent import FaraAgent
from test_framework.agents.magentic_executor import MagenticExecutor

class TestFlowGraph:
    """测试流程状态机"""

    def __init__(self):
        self.fara = FaraAgent()
        self.executor = MagenticExecutor()
        self.builder = StateGraph(TestState)
        self._build_graph()

    def _build_graph(self):
        # 定义节点
        self.builder.add_node("initialize", self._init_node)
        self.builder.add_node("explore", self._explore_node)
        self.builder.add_node("execute", self._execute_node)
        self.builder.add_node("verify", self._verify_node)
        self.builder.add_node("handle_defect", self._defect_node)
        self.builder.add_node("generate_report", self._report_node)

        # 定义边
        self.builder.set_entry_point("initialize")
        self.builder.add_edge("initialize", "explore")
        self.builder.add_edge("explore", "execute")
        self.builder.add_edge("execute", "verify")

        # 条件分支
        self.builder.add_conditional_edges(
            "verify",
            self._decide_next_step,
            {
                "continue": "explore",
                "defect": "handle_defect",
                "complete": "generate_report",
                "retry": "execute"
            }
        )

        self.builder.add_conditional_edges(
            "handle_defect",
            self._decide_after_defect,
            {
                "continue": "explore",
                "abort": "generate_report"
            }
        )

        self.builder.add_edge("generate_report", END)

        # 编译图（启用检查点）
        self.graph = self.builder.compile(checkpointer=MemorySaver())

    def _init_node(self, state: TestState) -> TestState:
        """初始化测试环境"""
        # 启动浏览器，加载初始页面
        screenshot = self.executor.init_browser(state["target_url"])
        return {
            **state,
            "screenshots": [screenshot],
            "current_screenshot": screenshot,
            "status": "initialized",
            "current_step": 0
        }

    def _explore_node(self, state: TestState) -> TestState:
        """Fara-7B分析页面并决策"""
        decision = self.fara.analyze_page(
            screenshot=state["current_screenshot"],
            history=state["actions"],
            goal=state["test_goal"]
        )

        return {
            **state,
            "planned_actions": [decision],
            "status": "exploring"
        }

    def _execute_node(self, state: TestState) -> TestState:
        """执行计划的操作"""
        action = state["planned_actions"][0]

        # 检查是否需要人工介入（基于Magentic-UI的HIL机制）
        if self._is_critical_action(action):
            # 触发人工确认
            pass

        result = self.executor.execute_action(action, state["current_screenshot"])

        return {
            **state,
            "actions": [{**action, **result}],
            "current_screenshot": result["new_screenshot"],
            "screenshots": [result["new_screenshot"]],
            "status": "executing",
            "current_step": state["current_step"] + 1
        }

    def _verify_node(self, state: TestState) -> TestState:
        """验证执行结果"""
        last_action = state["actions"][-1]
        verification = self.fara.verify_result(
            before_screenshot=state["screenshots"][-2] if len(state["screenshots"]) > 1 else None,
            after_screenshot=state["current_screenshot"],
            expectation=last_action.get("expected_result", "")
        )

        return {
            **state,
            "verification_results": [verification],
            "status": "verifying"
        }

    def _decide_next_step(self, state: TestState) -> str:
        """决定下一步状态"""
        last_verification = state["verification_results"][-1]

        if state["current_step"] >= 50:  # 最大步数限制
            return "complete"
        elif last_verification["is_success"]:
            return "continue"
        elif last_verification["is_defect"]:
            return "defect"
        elif last_verification["can_retry"]:
            return "retry"
        else:
            return "complete"

    def run(self, test_config: Dict) -> Dict:
        """执行测试流程"""
        initial_state = TestState(
            test_id=test_config["test_id"],
            target_url=test_config["url"],
            test_goal=test_config["goal"],
            current_step=0,
            status="pending",
            screenshots=[],
            actions=[],
            planned_actions=[],
            verification_results=[],
            defects=[],
            metadata={}
        )

        # 执行图
        final_state = self.graph.invoke(
            initial_state,
            config={"configurable": {"thread_id": test_config["test_id"]}}
        )

        return final_state
```

| **安全策略触发** | 检测到潜在危险操作（如下载文件、弹窗请求） | 拦截并请求人工确认 | 自动拒绝并记录 |
| **长时间无响应** | 页面加载或操作执行超过30秒 | 提示超时，询问是否继续等待 | 自动截图并尝试恢复 |

from typing import Callable, Optional
import asyncio

class HumanInTheLoopManager:
    """人机协同管理器"""

    def __init__(self, timeout: int = 300):
        self.timeout = timeout
        self.pending_confirmations = {}

    async def request_confirmation(
        self,
        context: Dict,
        callback: Optional[Callable] = None
    ) -> Dict:
        """
        请求人工确认

        Args:
            context: {
                "type": str,  # 触发类型
                "screenshot": bytes,  # 当前页面截图
                "description": str,  # 操作描述
                "risk_level": str,  # low/medium/high
                "suggested_action": str,  # 建议操作
                "alternatives": List[str]  # 替代方案
            }

        Returns:
            {
                "decision": str,  # "approve", "reject", "modify", "abort"
                "modified_action": Optional[Dict],  # 人工修改后的操作
                "comment": str  # 人工备注
            }
        """
        confirmation_id = self._generate_id()

        # 发送确认请求到前端界面
        await self._send_to_ui(confirmation_id, context)

        # 等待响应（带超时）
        try:
            response = await asyncio.wait_for(
                self._wait_for_response(confirmation_id),
                timeout=self.timeout
            )
            return response
        except asyncio.TimeoutError:
            return self._handle_timeout(context)

    def _handle_timeout(self, context: Dict) -> Dict:
        """超时默认处理策略"""
        if context["risk_level"] == "high":
            return {
                "decision": "reject",
                "comment": "超时自动拒绝高风险操作"
            }
        else:
            return {
                "decision": "approve",
                "comment": "超时自动批准低风险操作"
            }
